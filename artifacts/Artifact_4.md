
Подобрать подход для оценки каждого критерия и обосновать выбор.

а) Пользователь получает корректные значения КБЖУ 
Подход - измерение схожести с эталонными данными.
Этот критерий можно оценить без двусмысленности, так как существуют таблицы значений КБЖУ для различных продуктов с рассчитанными значениями. Использовать будем семантический подход с допустимой погрешность, тк даже в разных таблицах для одинаковых продуктов КБЖУ может немного отличаться.

б) Пользователь получает ответ быстро 
Подход - функциональная корректность.
Скорость ответа - объективно измеримая метрика. Будем сравнивать время ответа с заданным пороговым значением. 

в) Пользователь может вводить продукты естественным языком
Подход - AI-as-a-Judge
Этот критерий сложно точно оценить: могут быть синонимы (Сникерс можно назвать и шоколадкой, и батончиком, и конфетой), могут быть разные формулировки, может быть неявно указан вес (яблоко одно, пачка творога).
Так что сравнение с эталоном не подходит, функциональная проверка тоже нет. А модель-судья будет учитывать синонимы, формулировки и примерно понимать сколько весит продукт без явного указания веса.

г) Пользователь не получает в ответ вредный контент 
Подход - функциональная корректность и AI-as-a-Judge.
Этот критерий можно разбить на две части - на отлов конкретных стоп-слов (мат, оскорбления), которые можно отлавливать через регулярки (т.е. функциональная корректность) и на отлов более тонкой смысловой безопасности (вредные советы которые нельзя заложить на уровне ругулярок, тут нужна будет другая модель). 
Если модель-судья всё равно будет пропускать вредный контент, тогда либо дообучаем её, либо сажаем на проверку людей.
