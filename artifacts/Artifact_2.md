Артефакт - Декомпозиция AI-системы для оценки

(этот артефакт я делал на основе выполненного здания 1 - создание API-прототипа)

### 1. Декомпозировать систему с точки зрения того, какие задачи она решает, из каких компонентов состоит, а также из чего состоит выход системы.

а) Какие задачи решает система
1. Принять текстовый запрос пользователя с описанием продукта и его весе.
2. Распознать в запросе название продукта и его вес.
3. Сгенерировать структурированные данные о КБЖУ для этого продукта с учётом заданного веса.
4. Вернуть корректный JSON-ответ.

б) Компоненты системы
1. API-слой - принимает HTTP-запрос, валидирует входные данные, вызывает бизнес-логику, возвращает HTTP-ответ.
2. Слой промпт-инжиниринга - собирает системный промпт, задаёт правила и структуру ответа.
3. LLM-слой - получает сформированный промпт, обращается к LLM, генерирует текстовый ответ (в формате JSON).
4. Слой пост-обработки - получает ответ от LLM, обрабатывает возможные ошибки (чистит от возможно сгенерированного мусора), извлекает JSON, преобразует в объект.

в) Из чего состоит выход системы
Текстовый ответ в формате JSON, содержащий:
- Название продукта
- Количество (если продукт штучный)
- Единица измерения (граммы или шт)
- Вес (граммы)
- Калории (ккал)
- Белки (г)
- Жиры (г)
- Углеводы (г)

### 2. Среди задач, компонентов и частей выхода отобрать те, которые нуждаются в оценке.

а) Сквозной уровень
- Корректность получения ответа для нормальных запросов
- Обработка ошибок ввода и некорректных запросов (обработка запросов формата "кирпич 100 грамм" или "ыыыааа")

б) Промежуточный уровень:
- LLM-слой - самый важный компонент для оценки, тк является недетерминированным компонентом. Оцениваем структурную валидность сырого ответа, соответствие формату и корректность расчётов КБЖУ (приближённо).
