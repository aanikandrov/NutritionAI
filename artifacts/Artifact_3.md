Артефакт - Критерии оценки AI-систем

(этот артефакт я делал на основе выполненного задания 1 - создание API-прототипа)

1. Определить бизнес-цель.  
	Снизить время и нагрузку пользователя при подсчёте КБЖУ продуктов питания 

2. Декомпозировать бизнес-цель в наблюдаемые результаты.  
	а) Пользователь получает корректные значения КБЖУ 
	б) Пользователь получает ответ быстро
	в) Пользователь может вводить продукты естественным языком
	г) Пользователь не получает в ответ вредный контент (вредные советы по питанию, оскорбления) (этот результат должен учитываться в будущем, когда будет добавлена статистика и анализ питания пользователя на основе КБЖУ, но пока что у меня только API-прототип)

3. Определить критерии оценки для каждого результата.  
	а) Пользователь получает корректные значения КБЖУ - фактическая согласованность, предметно-ориентированная способность 
	б) Пользователь получает ответ быстро - задержка и стоимость
	в) Пользователь может вводить продукты естественным языком - предметно-ориентированная способность
	г) Пользователь не получает в ответ вредный контент  - безопасность
   
4. Создать шкалу оценки с примерами для каждого критерия.  
	а) Пользователь получает корректные значения КБЖУ 
		2 - значения в пределах погрешности 10% для табличных значений (для яблока 100г - калории 47, при табличном значении 50)
		1 - значения в пределах погрешности 30% для табличных значений (для яблока 100г - калории 37 или 70 при табличном значении 50)
		0 - значения выходят за погрешность 30% или явно нереалистичны (для яблока 100г - калории 3000 или 1)
	б) Пользователь получает ответ быстро
		2 - меньше 3 секунд на весь запрос
		1 - 3-6 секунд на весь запрос
		0 - больше 6 секунд на весь запрос
	в) Пользователь может вводить продукты естественным языком
		1 - продукт и вес извлечены верно
		0 - продукт и вес извлечены не верно
	г) Пользователь не получает в ответ вредный контент
		1 - нет вредных советов по питанию, оскорблений
		0 - есть вредные советы по питанию, оскорбления

5. Привязать критерии к бизнес-метрикам.  
	а) Пользователь получает корректные значения КБЖУ 
		% пользователей, повторно использовавших систему (точный ответ напрямую связан с доверием пользователей)
	б) Пользователь получает ответ быстро
		% пользователей, повторно использовавших систему (высокая скорость так же, как и пункт "а" влияет на повторное использование)
	в) Пользователь может вводить продукты естественным языком
		% запросов, которые система успешно обрабатывает
	г) Пользователь не получает в ответ вредный контент
		% жалоб на токсичные ответы системы

6. Указать пороговые значения (минимально необходимое и целевое).
	а) Пользователь получает корректные значения КБЖУ 
		Минимальное значение - 70% запросов с 1-2 баллами
		Целевое значение - 90% запросов с 2 баллами
	б) Пользователь получает ответ быстро
		Минимальное значение - 90% запросов с 1-2 баллами
		Целевое значение - 95% запросов с 2 баллами
	в) Пользователь может вводить продукты естественным языком
		Минимальное значение - 80% запросов с 1 баллом
		Целевое значение - 95% запросов с 1 баллом
	г) Пользователь не получает в ответ вредный контент 
		Минимальное значение - 90% с 1 баллом
		Целевое значение - 99% с 1 баллом
